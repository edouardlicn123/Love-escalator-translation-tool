# 翻译质量检查工具 - 项目计划

## 1. 项目概述

- **项目名称**: Translation QA Tool
- **目标**: 检查 `table.json` 的翻译质量，提供AI修正建议，支持交互式修正
- **输入文件**: `/home/edo/translation/table.json`
- **输出**: Web界面展示 + 修正后保存

---

## 2. 如何获取 HuggingFace API Token

### 步骤：

1. **注册账号**: 访问 https://huggingface.co 注册
2. **创建Token**:
   - 登录后点击右上角头像 → Settings → Access Tokens
   - 点击 "New token"
   - 填写名称（如 `translation-qa`）
   - 选择权限：**Read** 即可
   - 点击生成
3. **保存Token**: 复制生成的token，供程序使用

### 免费配额：
- MarianMT模型：**每月30000字符**（足够使用）

## 3. 功能规格

### 3.1 翻译质量检查（7项）

| # | 检查项 | 判定规则 |
|---|--------|----------|
| 1 | **cn为空** | cn字段为空字符串或仅空格 |
| 2 | **未翻译占位符** | 包含「」或……或... |
| 3 | **长度异常** | 中文字数 < 原文字数×0.3 或 > 原文字数×2.5 |
| 4 | **纯标点** | cn仅包含标点符号 |
| 5 | **数字占位** | 包含 `[NUM]` 或 `{n}` 等未替换占位符 |
| 6 | **相同内容** | cn与jp完全相同（未翻译） |
| 7 | **翻译质量低** | 包含"这个"、"那个"等机械翻译痕迹 |

### 3.2 AI翻译建议

- **API**: HuggingFace Inference API
- **模型**: `Helsinki-NLP/opus-mt-ja-zh`（日→中）
- **返回**: 参考翻译文本

### 3.3 Web界面功能

- 显示待修正条目列表
- 每条显示：原文(jp)、当前翻译(cn)、AI建议
- 复选框：勾选采纳AI建议
- 按钮：批量处理 / 上一条 / 下一条
- 自动保存：勾选后立即保存到 `table.json`
