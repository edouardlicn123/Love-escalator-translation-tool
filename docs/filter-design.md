# 过滤功能设计文档

## 需求概述

在搜索栏下方添加过滤选项，用于切换显示未确认/已确认的记录。

## UI 设计

```
┌─────────────────┐
│ 跳转            │
│ [输入框] [按钮] │
├─────────────────┤
│ 搜索            │
│ [输入框] [按钮] │
├─────────────────┤
│ [ ] 显示已确认  │
│ [确认]          │
└─────────────────┘
```

## 数据字段含义

| 字段 | 值 | 说明 |
|------|-----|------|
| is_fixed | 0 | 未确认 |
| is_fixed | 1 | 已确认 |

## 过滤逻辑

- **默认状态**：显示 `is_fixed = 0`（未确认）的记录
- **勾选后**：显示 `is_fixed = 1`（已确认）的记录

## API 设计

### GET /api/next

**参数** (URL query):
- `show_fixed`: 1 表示显示已确认的记录，不传或为0表示显示未确认

**示例**:
```
# 默认：显示未确认
/api/next

# 显示已确认
/api/next?show_fixed=1
```

## 实现清单

1. **后端** (`check_translation.py`):
   - 修改 `/api/next` 支持 `show_fixed` 参数

2. **前端** (`templates/index.html`):
   - 添加复选框和确认按钮
   - 修改调用逻辑传递参数
