## 一、后端优化计划

### 1.1 封装配置类

**问题**: 第6-17行定义了约12个全局变量，难以测试和维护。

**方案**: 创建 `AppConfig` dataclass 封装配置项。

**涉及位置**:
- PROXY_HOST, PROXY_PORT, PROXY_URL, PROXIES
- USE_PROXY, DATA_FILE, DB_FILE, SERVER_PORT
- LENGTH_RATIO_MIN, LENGTH_RATIO_MAX, CHECK_RULES

### 1.2 添加日志系统

**问题**: 整个代码没有日志，无法调试。

**方案**: 使用 `logging` 模块替代 print 语句。

**涉及位置**:
- 第120行配置加载失败
- 第414行服务启动信息

### 1.3 改进数据库连接

**问题**: 多处 `sqlite3.connect()` 未使用 `with` 语句，可能导致连接泄漏。

**方案**: 使用上下文管理器确保连接关闭。

**涉及位置**:
- 第48行 init_db()
- 第125行 get_db_stats()
- 第136行 sync_db()
- 第341行 do_GET 中的搜索逻辑
- 第393行 do_POST 中的更新逻辑

### 1.4 增强异常处理

**问题**: 第74行使用 `except Exception: pass` 静默吞掉所有异常。

**方案**: 添加日志记录或打印到 stderr。

**涉及位置**:
- load_groq_config() 第74行
- load_config() 第119行

### 1.5 优化 API 路由匹配

**问题**: 使用 `startswith()` 匹配 URL 精度不足。

**方案**: 使用 `urllib.parse.urlparse()` 解析路径后匹配。

**涉及位置**:
- 第313行 /api/next
- 第335行 /api/search

## 二、前端优化计划

### 2.1 统一变量作用域

**问题**: 第143-153行定义了 `state` 对象，但第376-388行又直接使用全局变量 (issues, currentIdx, showAll, useProxy 等)。

**方案**: 所有变量统一使用 `state` 对象管理。

**涉及位置**:
- 第376-388行 renderItem() 函数
- 第502-516行 prevItem(), nextItem() 函数

### 2.2 优化 DOM 更新

**问题**: 每次 renderItem() 都用 innerHTML 替换整个内容，性能较差。

**方案**: 使用事件委托，减少不必要的 DOM 操作。

**涉及位置**:
- 第185-200行 renderList()
- 第375-417行 renderItem()

### 2.3 添加加载状态 UI

**问题**: AI 翻译时只有文字 "(加载中...)"，无真实动画。

**方案**: 添加 CSS spinner 动画。

**涉及位置**:
- 第404行 AI建议区域

### 2.4 添加错误提示

**问题**: loadAISuggestion 失败时只打印 console，页面无反馈。

**方案**: 在页面上显示错误信息。

**涉及位置**:
- 第442-446行 错误处理逻辑

### 2.5 增强输入验证

**问题**: 第494行保存时只检查空字符串，未验证内容格式。

**方案**: 添加长度限制、特殊字符检查。

**涉及位置**:
- 第493-499行 saveAndNext()
